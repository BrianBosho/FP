# Codebase Refactor Plan (Phases A–D)

**Scope:** `federated-gnn/` repository  
**Primary goal:** make the codebase reproducible and maintainable by separating **source** from **generated artifacts**, standardizing entrypoints/config/paths, and enabling a safe package layout refactor with **backward compatibility**.

---

## Guiding principles

- **Source vs generated must be explicit**
  - Source (tracked): `src/`, `configs/` (or `conf/`), `scripts/`, `docs/`, `notebooks/` (inputs only).
  - Generated (untracked): `wandb/`, `runs/`, `results/`, `logs/`, caches, checkpoints, `.out`, etc.
- **No machine-specific paths**
  - Replace absolute paths (e.g., `/home/...`) with repo-relative resolution.
- **Backward compatible transitions**
  - Preserve existing commands like `python -m src.experiments.run_experiments ...` while introducing new structure.
  - When changing interfaces, prefer adding a “v2” path or config key and keep v1 working.
- **Minimize blast radius**
  - Phase changes should be independently shippable and reversible.

---

## Current observed issues (why this plan exists)

- **Artifacts were historically tracked** (e.g., `wandb/` run files). This bloats history and causes noisy diffs.
- **Outputs appear in unexpected locations**
  - `src/experiments/run_experiments.py` defaults `results_dir` to `../results/experiments`, which writes *outside* the repo when run from `federated-gnn/`.
- **Git ignore patterns were previously too broad**
  - Ignoring all `*.json`, `*.csv`, `*.txt`, etc. can accidentally hide files you *do* want to track (configs, docs, small fixtures).

## Phase A — Repo hygiene + reproducibility (no code moves)

### Objectives
- Ensure local runs don’t pollute the repo (or parent directories).
- Make environment setup reproducible for a new machine.
- Keep output directories consistent and predictable.

### Concrete tasks

#### A1) Standardize “generated output roots”
Decide on exactly one “run root” and one “data root” inside the repo:
- **Run root (generated):** `runs/` (recommended)
- **Results exports (generated):** `results/` (optional; treat as generated by default)
- **Data root (generated):** `data/` or `datasets/` (choose one canonical root; keep the other as a symlink/alias if needed)

#### A2) Make default output paths repo-local
Update defaults so they write inside `federated-gnn/`:
- Replace default `../results/experiments` with something like:
  - `runs/experiments` (recommended), or
  - `results/experiments`

**Backward compatibility:** still respect:
- `--results_dir ...`
- YAML `results_dir: ...`
- optional env var override (e.g., `FEDGNN_RUNS_DIR`)

#### A3) Remove absolute paths from code
Replace any machine-specific hardcoded defaults with repo-relative discovery:
- Example: `src/run.py` currently defaults to an absolute `conf/base.yaml` path; change to locate `conf/base.yaml` relative to the repo root.

#### A4) Add/confirm dependency management
Pick one and make it match `README.md`:
- **Option 1 (simple):** `requirements.txt` + optional `requirements-dev.txt`
- **Option 2 (recommended long-term):** `pyproject.toml` (packaging + tooling)
- If you prefer conda: add `environment.yml` and document a canonical conda env name.

### Acceptance criteria (“done”)
- Running experiments produces artifacts only under `runs/` (and/or `results/`) inside `federated-gnn/`.
- No code relies on `/home/...` absolute paths.
- A clean clone can install dependencies with one documented command.

## Phase B — Stop tracking generated artifacts (git cleanup)

### Objectives
- Ensure `git status` stays clean after experiments.
- Keep history and diffs small by tracking only source/config/docs.

### Concrete tasks

#### B1) `.gitignore` hardening (artifact directories + patterns)
Keep ignore rules **directory/pattern focused**, not “ignore whole extensions globally”.

Must ignore:
- `wandb/`
- `runs/`
- `results/`
- `results_summary/`
- `logs/`
- `training_logs/`
- `*.out`, `nohup*.out`
- model/checkpoint artifacts (`*.pt`, `*.pth`, `*.ckpt`, etc.)

#### B2) Remove already-tracked artifacts from the index
If any generated directories/files are tracked already:
- remove them from the index (while leaving local copies intact)
- commit the removal

#### B3) Optional: add a “clean” script
Add `scripts/clean_artifacts.sh` (or python equivalent) to delete local generated artifacts safely:
- only deletes known generated dirs (e.g., `wandb/`, `runs/`, `logs/`)
- never deletes configs/source
- supports `--dry-run`

### Acceptance criteria (“done”)
- After running experiments, generated files do not show up as untracked changes (they are ignored).
- Repo remains lightweight; no more committing W&B or run outputs.

## Phase C — Package + module refactor (structured code, keep old entrypoints working)

### Objectives
- Convert `src/` from “folder of scripts” to a proper, testable package layout.
- Improve discoverability: group by domain (`data`, `models`, `fl`, `experiments`, `analysis`, `utils`).
- Preserve existing module invocation patterns during transition.

### Proposed target structure (example)

- `src/fedgnn/`
  - `__init__.py`
  - `cli/` (optional)
  - `configs/` (optional, but configs usually stay outside `src/`)
  - `data/` (loaders, partitioning, datasets, feature propagation)
  - `fl/` (client/server/orchestration)
  - `models/`
  - `experiments/`
  - `analysis/`
  - `utils/`
- Keep legacy path:
  - `src/` can temporarily keep `run.py` and `experiments/run_experiments.py` as thin wrappers importing from `fedgnn.*`

### Concrete tasks

#### C1) Introduce a real package namespace
Create `src/fedgnn/` and begin moving code into it in small chunks.

#### C2) Add compatibility shims
Keep these working:
- `python -m src.experiments.run_experiments --config conf/...`

Approach:
- keep `src/experiments/run_experiments.py` as wrapper that calls `fedgnn.experiments.run_experiments:main()`
- keep imports stable by re-exporting or aliasing moved functions/classes

#### C3) Standardize configuration + path utilities
Add a single “project paths” module (e.g., `fedgnn/utils/paths.py`) that:
- finds repo root
- resolves `conf/` (or `configs/`)
- resolves `runs/` output roots
- avoids `..` path usage

#### C4) Add lightweight tests for critical invariants (optional but recommended)
At minimum:
- config load/merge works
- run output directories are created correctly
- CLI parsing doesn’t break

### Acceptance criteria (“done”)
- New code lives under `src/fedgnn/` with clean imports.
- Old CLI/module entrypoints still run without changes.
- Path resolution is centralized and consistent.

---

## Phase D — Notebooks + analysis consolidation

### Objectives
- Make notebooks reproducible and keep them from generating tracked junk.
- Separate “analysis code” from “analysis outputs”.

### Concrete tasks

#### D1) Notebook policy
- Keep notebooks in `notebooks/` (tracked)
- Ensure notebook outputs go to `runs/` or `results/` (ignored)
- If notebooks need data, they should read from the canonical data root (`data/` or `datasets/`)

#### D2) Promote reusable analysis code into `src/fedgnn/analysis/`
Any parsing utilities used in notebooks (e.g., training log parsing) should be:
- importable python modules
- unit-tested if feasible

#### D3) Provide an “export” mechanism
If you want to keep certain final artifacts (paper-ready plots/tables):
- export to `reports/` (tracked) **only if intentionally curated**
- otherwise keep under `runs/` (ignored)

### Acceptance criteria (“done”)
- Notebooks are stable to run end-to-end and don’t write into tracked paths.
- Analysis logic is shared code, not copy/pasted per notebook.
- Generated plots/CSVs go to ignored output directories unless explicitly curated.

## Risks + mitigations

- **Risk:** breaking existing commands/imports during Phase C  
  - **Mitigation:** wrappers + re-exports; move code incrementally; keep `src/...` module paths alive until all consumers migrate.
- **Risk:** losing valuable run artifacts by “clean” scripts  
  - **Mitigation:** default to `--dry-run`, require explicit confirmation flags, only delete known generated dirs.
- **Risk:** configs/results confusion (tracked vs untracked)  
  - **Mitigation:** keep configs in `conf/` (tracked), keep outputs in `runs/` (ignored), document the convention in `README.md`.
